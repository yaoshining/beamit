### 用户故事 1 - 一键投屏到电视（优先级：P1）

用户在使用 Chrome 浏览网页时，发现网页上的视频想要投屏到电视观看。用户点击扩展图标，扩展自动检测页面中的视频流，列出可用的投屏设备，用户选择电视后视频立即在大屏播放。

**为什么这个优先级**：这是核心用户旅程，定义了产品的基本价值主张。

**独立测试**：可以通过模拟视频源和虚拟投屏设备完全测试。

**验收场景**：

1. **给定**用户在播放视频的网页上，**当**点击扩展图标，**那么**扩展显示已检测到视频流
2. **给定**扩展已检测到视频流，**当**用户点击投屏按钮，**那么**显示局域网内的可用设备列表
3. **给定**用户已选择目标电视，**当**投屏进行中，**那么**显示正在播放状态和播放进度

---

### 用户故事 2 - 智能视频流检测（优先级：P2）

用户访问一个包含多个视频源的复杂网页，扩展能够自动识别并列出所有可用的视频流，用户可以选择其中一个进行投屏。

**为什么这个优先级**：增强产品可用性，支持更多视频源场景。

**独立测试**：可以通过注入测试视频元素到页面来验证检测能力。

**验收场景**：

1. **给定**网页包含 m3u8 格式视频，**当**用户打开扩展，**那么**检测并显示该视频源
2. **给定**网页包含多个视频源，**当**用户打开扩展，**那么**显示所有可用视频的列表
3. **给定**网页没有可检测的视频，**当**用户打开扩展，**那么**显示"未检测到可投屏视频"的提示

---

### 用户故事 3 - 投屏设备管理（优先级：P3）

用户可以在扩展中查看最近使用过的投屏设备，并快速重新连接常用设备。

**为什么这个优先级**：提升用户效率，减少重复操作。

**独立测试**：可以通过模拟设备历史记录来测试。

**验收场景**：

1. **给定**用户之前使用过某些设备，**当**打开扩展，**那么**显示最近使用过的设备列表
2. **给定**用户在设备列表中点击某个设备，**当**设备在线，**那么**立即发起投屏请求
3. **给定**用户点击某个离线设备，**那么**显示设备离线提示

## 需求（必填）

### 功能需求

- **FR-001**：系统必须能够检测网页中的主流视频流格式（m3u8、mp4、webm、flv）
- **FR-002**：用户必须能够在扩展弹出窗口中查看检测到的视频源列表
- **FR-003**：系统必须能够在局域网内发现支持 DLNA 协议的设备
- **FR-004**：用户必须能够通过点击设备列表来发起投屏请求
- **FR-005**：系统必须显示投屏状态（正在连接、正在播放、已断开）
- **FR-006**：用户必须能够随时中断投屏
- **FR-007**：系统必须记住用户最近使用的投屏设备

### 关键实体

- **视频源**：代表网页中检测到的视频流，包含 URL、视频类型、时长、是否可播放
- **投屏设备**：代表局域网内的可用设备，包含名称、设备类型、协议类型、地址、在线状态
- **投屏会话**：代表一次投屏操作，包含状态、开始时间、目标设备、源视频信息

### 视频检测策略（详细）

1. **Media 元素检测**：遍历页面中所有 `<video>` 和 `<audio>` 标签，提取 src 和 currentSrc
2. **HLS/m3u8 检测**：解析页面中所有 script 标签和 manifest 请求，寻找 .m3u8 URL
3. **DASH/mpd 检测**：解析 m3u8 内容，检测是否有 .mpd 或自适应流信息
4. **Iframe 检测**：检测内嵌的播放器 iframe（如 YouTube、Vimeo 等）

### DLNA 实现要点

1. **设备发现**：使用 SSDP (Simple Service Discovery Protocol) 在局域网内发现 DMR (Digital Media Renderer) 设备
2. **设备描述**：获取设备的 SOAP XML 描述文件
3. **媒体控制**：使用 AVTransport 和 RenderingControl 服务控制播放
4. **协议映射**：HTTP/HTTPS URL 直接传给设备

### 数据流

```
用户点击扩展图标
    ↓
Popup 打开，发送消息给 Service Worker
    ↓
Service Worker 请求 Content Script 检测视频
    ↓
Content Script 返回视频源列表
    ↓
Popup 显示视频列表 + 设备列表
    ↓
用户选择设备后，Service Worker 发起 DLNA 播放请求
    ↓
DLNA 设备开始播放
```
